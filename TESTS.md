## 2. SEO-тесты

### 2.1 Мета-теги

- [ ] Каждая страница имеет уникальный `<title>` (до 60 символов)
- [ ] Каждая страница имеет уникальный `<meta description>` (150-160 символов)
- [ ] `<link rel="canonical">` присутствует на каждой странице
- [ ] canonical URL правильный (без дубликатов)
- [ ] `<meta name="robots">` корректен (index/noindex)
- [ ] hreflang теги присутствуют для всех языковых версий
- [ ] `<html lang="ru">` установлен
- [ ] viewport мета-тег присутствует

### 2.2 Open Graph

- [ ] og:title присутствует
- [ ] og:description присутствует
- [ ] og:image присутствует и URL валиден
- [ ] og:image размер 1200x630px
- [ ] og:url правильный
- [ ] og:type корректен (website/article)
- [ ] og:locale установлен (ru_RU)
- [ ] article:published_time для статей
- [ ] article:modified_time для статей

### 2.3 Twitter Card

- [ ] twitter:card присутствует (summary_large_image)
- [ ] twitter:title присутствует
- [ ] twitter:description присутствует
- [ ] twitter:image присутствует

### 2.4 Structured Data (JSON-LD)

- [ ] Organization schema на главной
- [ ] WebSite schema с SearchAction на главной
- [ ] Article schema на страницах статей
- [ ] BreadcrumbList schema на всех вложенных страницах
- [ ] FAQPage schema на страницах с FAQ
- [ ] HowTo schema на пошаговых инструкциях
- [ ] DefinedTerm schema в глоссарии
- [ ] Все schema проходят валидацию (Google Rich Results Test)
- [ ] Нет ошибок/предупреждений в Google Search Console

### 2.5 Sitemap и Robots

- [ ] sitemap.xml доступен по URL
- [ ] sitemap содержит все публичные страницы
- [ ] sitemap не содержит noindex-страницы
- [ ] lastmod даты в sitemap актуальны
- [ ] robots.txt доступен
- [ ] robots.txt ссылается на sitemap
- [ ] robots.txt не блокирует нужные страницы
- [ ] robots.txt блокирует служебные страницы

### 2.6 Контентное SEO

- [ ] H1 заголовок единственный на каждой странице
- [ ] Иерархия заголовков H1→H2→H3 корректна (без пропусков)
- [ ] Ключевое слово в H1
- [ ] Ключевое слово в первых 100 словах
- [ ] Ключевое слово в URL
- [ ] Все изображения имеют alt-текст
- [ ] Внутренние ссылки используют осмысленный anchor text (не "нажмите сюда")
- [ ] Нет дублирующегося контента
- [ ] Нет тонкого контента (< 300 слов без причины)

### 2.7 Техническое SEO

- [ ] Сайт доступен по HTTPS
- [ ] HTTP → HTTPS редирект работает
- [ ] www → non-www редирект (или наоборот) — единый формат
- [ ] Trailing slash единообразный
- [ ] 404 страница кастомная и полезная
- [ ] 404 страница возвращает HTTP 404
- [ ] Нет цепочек редиректов (A→B→C)
- [ ] Нет битых ссылок (внутренних и внешних)
- [ ] JavaScript не блокирует рендеринг
- [ ] CSS не блокирует рендеринг критических элементов

## 3. Тесты производительности

### 3.1 Core Web Vitals

- [ ] **LCP** (Largest Contentful Paint) < 2.5 секунды
- [ ] **INP** (Interaction to Next Paint) < 200 мс
- [ ] **CLS** (Cumulative Layout Shift) < 0.1
- [ ] **FCP** (First Contentful Paint) < 1.8 секунды
- [ ] **TTFB** (Time to First Byte) < 800 мс
- [ ] **TBT** (Total Blocking Time) < 200 мс

### 3.2 Lighthouse Score

- [ ] Performance: > 90 (мобильный)
- [ ] Performance: > 95 (десктоп)
- [ ] Accessibility: > 95
- [ ] Best Practices: > 95
- [ ] SEO: > 95

### 3.3 Размер страницы

- [ ] HTML < 100 KB (gzip)
- [ ] CSS < 50 KB (gzip)
- [ ] JS < 100 KB (gzip) — для статических страниц
- [ ] Общий вес страницы < 500 KB (без изображений)
- [ ] Количество HTTP-запросов < 30

### 3.4 Изображения

- [ ] Используется WebP/AVIF формат
- [ ] Responsive images (srcset)
- [ ] Lazy loading для изображений ниже первого экрана
- [ ] Заданы width и height (предотвращает CLS)
- [ ] Изображения оптимизированы (сжаты)
- [ ] OG-изображения: 1200x630, < 300 KB

### 3.5 Кэширование

- [ ] Статические ассеты имеют Cache-Control: immutable, max-age=31536000
- [ ] HTML страницы: Cache-Control: public, max-age=3600 (или revalidate)
- [ ] Service Worker (опционально, для offline)
- [ ] CDN кэширует корректно

### 3.6 Нагрузочные тесты

- [ ] Сайт выдерживает 100 одновременных пользователей
- [ ] Сайт выдерживает 1000 одновременных пользователей
- [ ] Время ответа < 200ms при нагрузке
- [ ] Нет утечек памяти при длительной сессии
